# Prioritisation and Decision Tools

Tools for ranking options, prioritising requirements or recommendations, and making transparent, auditable decisions. These tools appear in methodology sections (options appraisal, recommendation development) and in work plans (phasing decisions).

---

## Prioritisation Techniques

### MoSCoW

| Category | Meaning | Guidance |
|---|---|---|
| **Must have** | Critical for success; solution fails without it | Non-negotiable requirements |
| **Should have** | Important but not vital; workaround exists | High priority, second tier |
| **Could have** | Desirable but not necessary | Include if time/budget permits |
| **Won't have** (this time) | Out of scope for current release | Documented for future consideration |

**Proposal use:** Prioritise deliverables, features, or recommendations into phases. "Must-have" items go into Phase 1; "Should-have" into Phase 2.

### Weighted Ranking Matrix

Objective comparison of multiple alternatives against weighted criteria.

| Criterion | Weight (%) | Option A Rating | Option A Score | Option B Rating | Option B Score |
|---|---|---|---|---|---|
| [Criterion 1] | [%] | [1-5] | [Weighted] | [1-5] | [Weighted] |
| [Criterion 2] | [%] | [1-5] | [Weighted] | [1-5] | [Weighted] |
| **Total** | **100%** | | **[Sum]** | | **[Sum]** |

**Rules:**
- Select 3–9 criteria aligned with goals and objectives
- Assign weights by percentages or decimals (total = 100% or 1.0)
- Score each option against each criterion
- Multiply scores by weights; sum for overall ranking
- Verify results make sense to evaluators

**Proposal language:** "Options will be evaluated using a weighted criteria matrix agreed with the client, ensuring transparent and auditable decision-making."

### Multivoting

- Each participant receives a limited number of votes (typically N/3 where N = number of items)
- Votes applied to preferred options from a list
- Items with most votes are shortlisted
- Quick technique for narrowing a large set to a manageable shortlist

### Timeboxing

- Prioritise as much work as possible within a predetermined time frame or budget
- Items that do not fit are deferred to the next timebox
- Forces prioritisation by constraint rather than by preference

### T-Shirt Sizing

- Group items as Small, Medium, or Large based on defined criteria (effort, complexity, risk)
- Quick relative estimation technique
- Good for early-stage planning before detailed estimates

### Kano Model

Categorises features by their relationship to customer satisfaction:

| Category | Description | Implication |
|---|---|---|
| **Must-be** | Expected baseline; absence causes dissatisfaction | Must be included — non-negotiable |
| **Performance** | Linear relationship — more is better | Prioritise by value/cost ratio |
| **Attractive** | Delight factors; absence does not cause dissatisfaction | Differentiators — include strategically |

### Priority Matrix (Business Value × Implementation Difficulty)

| | Low Difficulty | High Difficulty |
|---|---|---|
| **High Value** | **HIGH PRIORITY** — do first | Medium priority (check frequency of use) |
| **Low Value** | Medium priority (check frequency of use) | **LOW PRIORITY** — avoid or defer |

Medium-priority features can be relabelled as high priority if they have high frequency of use.

## Decision-Making Tools

### Multi-Criteria Decision Analysis (MCDA)

Structured, auditable framework for comparing options.

| Criterion | Weight (%) | Option A | Score A | Option B | Score B | Option C | Score C |
|---|---|---|---|---|---|---|---|
| [Criterion 1] | [%] | [Rating] | [Weighted] | [Rating] | [Weighted] | [Rating] | [Weighted] |
| **Total** | **100%** | | **[Sum]** | | **[Sum]** | | **[Sum]** |

**Rules:** Criteria and weights agreed with the client before scoring. Transparent and auditable.

**Proposal language:** "We will apply Multi-Criteria Decision Analysis (MCDA) with criteria and weights agreed with the Steering Committee to ensure objective, transparent evaluation."

### Impact-Effort Matrix (Prioritisation Matrix)

| | Low Effort | High Effort |
|---|---|---|
| **High Impact** | **Quick wins** — do first | **Major projects** — plan carefully |
| **Low Impact** | **Fill-ins** — do if time permits | **Avoid** — low return on investment |

**Deloitte pattern:** Compile a "catalogue of opportunities", then prioritise using a criteria matrix before developing business cases for the top-priority items.

### Options Appraisal

Standard structure for any assignment requiring a recommended course of action:

1. **Define the options** — including "do nothing" as a baseline
2. **Define evaluation criteria** — agreed with the client
3. **Assess each option** against the criteria
4. **Present the comparison** with a recommended option and rationale

**Proposal language:** "We will develop a minimum of three options (including a 'do nothing' baseline) and evaluate each against agreed criteria covering technical feasibility, cost, risk, and alignment with [client] strategy."

### Force Field Analysis

Analyses driving forces (for change) and restraining forces (against change):

| Driving Forces (For) | Strength | Restraining Forces (Against) | Strength |
|---|---|---|---|
| [Force 1] | [H/M/L] | [Force 1] | [H/M/L] |

**Use:** Understand what influences a situation; identify which restraining forces to weaken and which driving forces to strengthen.

### Delphi Technique

- Anonymous iterative polling to reach consensus
- Experts submit opinions independently
- Results aggregated and shared; experts revise
- Repeat until convergence
- Removes influence of dominant personalities

## Financial Decision Tools

### Cost-Benefit Analysis (CBA)

| Item | Year 1 | Year 2 | Year 3 | Year 4 | Year 5 | Total |
|---|---|---|---|---|---|---|
| **Benefits** | | | | | | |
| **Costs** | | | | | | |
| **Net benefit** | | | | | | |
| **NPV** | | | | | | |
| **BCR** | | | | | | |

Include discount rate, Net Present Value (NPV), and Benefit-Cost Ratio (BCR). State the discount rate used and why.

### Financial Evaluation Methods

| Method | Description | Decision Rule |
|---|---|---|
| **Payback Period (PBP)** | Time to recover project investment | Shorter = less risk |
| **Return on Investment (ROI)** | (Net Benefits / Total Costs) × 100 | Higher = better return |
| **Internal Rate of Return (IRR)** | Projected annual yield including initial and ongoing costs | Compare against hurdle rate |
| **Net Present Value (NPV)** | Future benefits expressed in present-day value | Any NPV > 0 is worthwhile |

Always present ROI alongside payback period — how many months until the investment pays for itself.

### Total Cost of Ownership (TCO)

- Capital expenditure (hardware, software, implementation)
- Recurring costs (licences, hosting, maintenance, support staff)
- Hidden costs (training, change management, downtime, opportunity cost)
- Compare across options over a five-year horizon

**Proposal language:** "Our financial analysis will include a five-year TCO comparison across all options, covering capital, recurring, and hidden costs."

## Group Decision-Making Approaches

| Approach | Description | When to Use |
|---|---|---|
| **Unanimous agreement** | Everyone agrees | Ideal but rare for complex decisions |
| **Majority vote** | More than half agrees | Quick decisions with clear options |
| **Consensus** | Everyone feels they had fair chance to influence | Important decisions needing broad buy-in |
| **Expert decision** | Specialist makes the call | Technical decisions requiring domain expertise |
| **Authority decision** | Designated decision-maker decides | Time-critical or governance-mandated decisions |
