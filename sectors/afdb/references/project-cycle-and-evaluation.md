# AfDB Project Cycle and Evaluation

Reference material for the afdb sector skill. Covers the AfDB project cycle, Project Completion Report (PCR) structure, Independent Development Evaluation (IDEV) framework, OECD-DAC evaluation criteria, the 4-point rating scale, sustainability assessment, and the Project Assessment Report (PAR) template.

Sources: AfDB Operations Manual, IDEV Evaluation Policy, IDEV Evaluation Guidelines, AfDB PCR Guidelines, OECD-DAC Evaluation Criteria (2019 revision).

---

## 1. AfDB Project Cycle — Detailed

### 1.1 Country Strategy Paper (CSP)

- 5-year strategy document for each borrowing RMC
- Aligned with the country's national development plan
- Identifies priority sectors for AfDB investment
- Includes a lending pipeline and indicative resource allocation
- Mid-term review after 2-3 years

**Consultant opportunity:** Country diagnostics, sector studies, institutional assessments that feed into CSP preparation.

### 1.2 Project Identification

- Concept Note prepared by the borrower or AfDB sector department
- Initial screening for E&S risks (project categorisation)
- Preliminary cost estimate
- Project added to the lending pipeline

### 1.3 Project Preparation

- Detailed feasibility study
- ESIA (for Category 1 and 2)
- Engineering design and cost estimates
- Financial and economic analysis
- Institutional and implementation arrangements
- Procurement plan prepared
- Project Appraisal Report (PAR) drafted

**Consultant opportunity:** This is the peak phase for consulting services — feasibility studies, ESIAs, design, institutional analysis, cost estimation.

### 1.4 Project Appraisal

- AfDB appraisal mission reviews all preparation work
- Technical, economic, financial, institutional, E&S, and gender assessments
- Negotiations with borrower on loan/grant terms, conditions, and covenants
- Board presentation document (PAR) finalised

### 1.5 Board Approval

- The Board of Directors approves the loan/grant
- Conditions precedent to first disbursement specified
- Loan/grant declared effective when conditions met

### 1.6 Implementation

- Borrower procures goods, works, and services per the approved procurement plan
- AfDB supervision through implementation support missions (typically 2 per year)
- Mid-Term Review (MTR) conducted midway through project life
- Key milestone: Results framework monitored, implementation progress reports reviewed

**Consultant opportunity:** Implementation support, supervision, capacity building, MTR evaluation, procurement support.

### 1.7 Project Completion

- Project Completion Report (PCR) prepared jointly by the borrower and AfDB
- Final disbursement and financial closure
- Preliminary assessment of outcomes and sustainability

### 1.8 Post-Completion Evaluation

- Independent Development Evaluation (IDEV) may select projects for post-completion evaluation
- PCR validation — IDEV reviews the PCR for accuracy and self-assessment quality
- Project Performance Evaluation Report (PPER) — deeper independent evaluation of selected projects

## 2. Project Completion Report (PCR) Structure

The PCR is the primary tool for learning from completed projects. Structure:

### Part A: Project Data and Key Dates

- Project name, sector, country, approval date, effectiveness date, completion date
- Planned vs actual costs (by component and financing source)
- Key milestone dates (planned vs actual)
- Disbursement profile

### Part B: Project Performance

**B.1 Relevance**
- Relevance of project objectives to country strategy
- Relevance of project design to objectives
- Rating: 1-4 scale

**B.2 Effectiveness**
- Achievement of expected outputs
- Achievement of expected outcomes
- Rating: 1-4 scale

**B.3 Efficiency**
- Economic rate of return (ERR) vs appraisal estimate
- Implementation timeliness (planned vs actual completion)
- Cost efficiency (planned vs actual costs)
- Rating: 1-4 scale

**B.4 Sustainability**
- Financial sustainability
- Institutional sustainability and capacity
- Environmental and social sustainability
- Ownership and stakeholder support
- Rating: 1-4 scale

### Part C: Performance of Stakeholders

- Borrower performance (preparation, implementation, compliance with covenants)
- AfDB performance (identification, preparation, supervision, responsiveness)
- Rating: 1-4 for each

### Part D: Cross-Cutting Issues

- Gender and social inclusion outcomes
- Climate change and environmental outcomes
- Private sector development outcomes
- Knowledge and capacity building outcomes

### Part E: Lessons Learned and Recommendations

- Key lessons (what worked, what did not, and why)
- Specific recommendations for future operations
- Recommendations for the borrower

## 3. IDEV — Independent Development Evaluation

IDEV is AfDB's independent evaluation department, reporting directly to the Board. It operates independently from AfDB Management.

### 3.1 IDEV Mandate

- Evaluate the development effectiveness and institutional performance of AfDB operations
- Provide evidence-based findings, lessons, and recommendations
- Foster a culture of learning and accountability

### 3.2 IDEV Evaluation Types

| Type | Scope | Timing |
|---|---|---|
| **PCR Validation** | Desk review of PCR quality and self-assessment accuracy | Within 1 year of PCR |
| **Project Performance Evaluation Report (PPER)** | In-depth independent evaluation of individual projects | 3-5 years after completion |
| **Country Strategy Evaluation** | Evaluation of AfDB's strategy and portfolio in a country | Every 5-10 years per country |
| **Sector/Thematic Evaluation** | Cross-cutting evaluation of a sector or theme across countries | As needed |
| **Corporate Evaluation** | Evaluation of AfDB-wide policies, strategies, or processes | As needed |
| **Impact Evaluation** | Rigorous assessment of causal impact using experimental/quasi-experimental methods | Selected projects |
| **Cluster Evaluation** | Evaluation of a group of related projects | As needed |
| **Synthesis Report** | Meta-analysis of evaluation findings across a portfolio | Periodic |

### 3.3 IDEV Independence

- Reports to the Board of Directors, not Management
- Director General of IDEV appointed by the Board
- IDEV sets its own work programme
- Management provides a formal response to evaluation findings
- Board discusses evaluations and Management responses

## 4. OECD-DAC Evaluation Criteria

AfDB uses the OECD-DAC evaluation criteria (2019 revision) as the standard framework for all evaluations:

### 4.1 Relevance

*Is the intervention doing the right things?*

- Extent to which objectives and design respond to beneficiaries', country, and partner/institution needs, policies, and priorities
- Extent to which the intervention responds to the evolving context
- Alignment with AfDB strategy and RMC priorities

**Rating questions:**
- Were the objectives appropriate given the context?
- Was the design appropriate for achieving the objectives?
- Did the intervention remain relevant as the context evolved?

### 4.2 Coherence

*How well does the intervention fit?*

- Internal coherence: alignment between objectives, design, implementation
- External coherence: consistency with other interventions in the same context
- Complementarity with other development partners' activities
- Added value of AfDB's contribution relative to others

**Rating questions:**
- Is the intervention consistent with other AfDB operations in the same country/sector?
- Does it complement or duplicate what other development partners are doing?
- Is there a clear added value from AfDB's involvement?

### 4.3 Effectiveness

*Is the intervention achieving its objectives?*

- Achievement of stated outcomes and outputs
- Contribution to higher-level development goals
- Unintended effects (positive and negative)
- Differential effects across groups (gender, age, income, geography)

**Rating questions:**
- Were the expected outputs delivered?
- Were the expected outcomes achieved?
- Were there significant unintended effects?

### 4.4 Efficiency

*How well are resources being used?*

- Cost-efficiency: cost of inputs relative to outputs
- Timeliness: implementation within planned schedule
- Economic efficiency: economic rate of return
- Comparison with alternative approaches

**Rating questions:**
- Was the intervention delivered on time and within budget?
- Could the same results have been achieved at lower cost?
- Was the economic return adequate?

### 4.5 Impact

*What difference does the intervention make?*

- Significant positive and negative, primary and secondary long-term effects
- Direct and indirect effects
- Intended and unintended effects
- Systemic and transformational effects

**Rating questions:**
- What has happened as a result of the intervention?
- What real difference has it made to the beneficiaries?
- How many people have been affected?

### 4.6 Sustainability

*Will the benefits last?*

- Financial sustainability — continued funding or revenue generation after AfDB support ends
- Institutional sustainability — capacity and systems to maintain outcomes
- Environmental sustainability — no environmental degradation from continued operation
- Social sustainability — continued social acceptance and equity
- Technical sustainability — ability to maintain technologies and systems
- Ownership — government and stakeholder commitment to sustaining outcomes

**Rating questions:**
- Are the benefits likely to continue after the project ends?
- Is there adequate institutional capacity to sustain outcomes?
- Are there financial mechanisms to sustain operations?
- Do stakeholders have ownership and commitment?

## 5. The 4-Point Rating Scale

IDEV uses a 4-point rating scale for all evaluation criteria:

| Score | Rating | Definition |
|---|---|---|
| **4** | Highly Satisfactory | Exceeds expectations; outstanding performance with no significant shortcomings |
| **3** | Satisfactory | Meets expectations; good performance with only minor shortcomings |
| **2** | Unsatisfactory | Below expectations; significant shortcomings that undermine performance |
| **1** | Highly Unsatisfactory | Well below expectations; severe shortcomings; no meaningful positive results |

**Overall project rating:**
- Weighted average of the six criteria ratings
- An overall rating of 3 or above = satisfactory project
- Below 3 = unsatisfactory

**Rating distribution (AfDB portfolio, typical):**
- Approximately 70-75% of completed projects rated satisfactory or above
- Agriculture and social sector projects tend to rate lower on sustainability
- Infrastructure projects tend to rate higher on efficiency but face sustainability challenges

## 6. Sustainability Assessment — 8 Sub-factors

IDEV evaluates sustainability across 8 sub-factors:

| Sub-factor | Assessment Focus |
|---|---|
| **Financial sustainability** | Revenue generation, cost recovery, budget allocation for O&M |
| **Institutional sustainability** | Organisational capacity, staffing, management systems |
| **Technical sustainability** | Ability to maintain equipment, technology, and systems |
| **Environmental sustainability** | Long-term environmental impacts and resource management |
| **Social sustainability** | Continued social acceptance, equity, community engagement |
| **Political sustainability** | Government commitment, policy continuity |
| **Economic sustainability** | Continued economic viability, market conditions |
| **Ownership** | Stakeholder commitment to sustaining outcomes |

**Proposal implication:** When writing methodology or sustainability sections for AfDB-funded assignments, address all 8 dimensions. This demonstrates understanding of what AfDB evaluates at completion.

## 7. Project Assessment Report (PAR) / Appraisal Report Structure

The PAR is the core document presented to the Board for approval. Understanding its structure helps consultants prepare inputs:

### Executive Summary
- Project overview, rationale, expected outcomes, costs, financing plan

### Strategic Context
- Country context and CSP alignment
- Sector context and challenges
- AfDB experience in the sector/country

### Project Description
- Project components and activities
- Cost estimates and financing plan
- Implementation arrangements
- Procurement plan summary

### Project Feasibility
- Technical feasibility
- Economic and financial analysis (ERR, NPV, sensitivity analysis)
- Environmental and social assessment summary
- Climate risk assessment
- Gender assessment

### Implementation
- Institutional arrangements
- Procurement arrangements
- Disbursement schedule
- Monitoring and evaluation plan
- Risk management

### Legal Framework
- Conditions precedent to effectiveness
- Conditions precedent to disbursement
- Other covenants (reporting, audit, environmental compliance)

### Recommendation
- Management recommendation to the Board

## 8. Evaluation Quality Standards

IDEV evaluations must meet quality standards:

**Design phase:**
- Clear evaluation questions aligned with OECD-DAC criteria
- Appropriate methodology (mixed methods preferred)
- Realistic scope given time and resources
- Stakeholder engagement plan

**Implementation phase:**
- Systematic data collection
- Triangulation of sources
- Gender-responsive evaluation approach
- Ethical standards (informed consent, confidentiality, do no harm)

**Reporting phase:**
- Evidence-based findings clearly linked to data
- Ratings justified with evidence
- Lessons specific and actionable
- Recommendations prioritised and addressed to specific actors

**Peer review:**
- Internal quality review by IDEV staff
- External peer review by independent experts for major evaluations

## 9. Management Response and Follow-Up

After each evaluation:

1. **Management Action Record (MAR):** AfDB Management provides a formal response accepting, partially accepting, or rejecting each recommendation
2. **Action Plan:** For accepted recommendations, Management prepares an implementation action plan with timelines and responsibilities
3. **Follow-up:** IDEV tracks implementation of accepted recommendations
4. **Board review:** The Board's Committee on Operations and Development Effectiveness (CODE) reviews evaluations and Management responses

**Implication for consultants:** When conducting evaluations for AfDB, structure recommendations so they are:
- Specific (who should do what)
- Actionable (can be implemented within existing constraints)
- Time-bound (clear implementation timeframe)
- Prioritised (critical vs desirable)

## 10. Lessons from AfDB Evaluation Portfolio

Common findings across AfDB evaluations that consultants should be aware of:

**Positive patterns:**
- Infrastructure projects generally deliver outputs on time
- AfDB supervision has improved with more frequent implementation support missions
- Projects with strong M&E frameworks perform better

**Recurring challenges:**
- **Sustainability** is the weakest-rated criterion across the portfolio
- Implementation delays (average project extends 1-2 years beyond original timeline)
- Insufficient attention to O&M in project design
- Weak institutional capacity of implementing agencies
- Inadequate counterpart funding from borrowing governments
- Gender mainstreaming often superficial (noted as a gap, not adequately resourced)

**Proposal implication:**
- Build sustainability considerations into every methodology from the start
- Include realistic timelines with contingency
- Address O&M explicitly in project design
- Propose concrete capacity building, not generic training
- Budget adequately for gender-responsive approaches
- Include adaptive management mechanisms to handle delays
