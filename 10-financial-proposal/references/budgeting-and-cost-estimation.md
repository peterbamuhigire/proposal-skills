# Budgeting and Cost Estimation Reference

Synthesised from: Bielefeld & Schneider (*Basics Budgeting*, 2014), Weiler & Serna (*Practical Strategies for Applied Budgeting and Fiscal Administration*, 2016), Gordon (*Be Smart About Money*, 2014), Lewis (*Project Planning, Scheduling & Control*, 6th ed.).

This reference serves two purposes:
1. **Writing financial proposals** — constructing credible, well-structured project budgets for consulting bids
2. **Consulting assignments involving budgeting** — advising clients on budget planning, fiscal administration, and cost management

---

## 1. Budget Allocation Methodologies

Three approaches to constructing budgets, applicable to both proposal budgets and client advisory work.

### 1.1 Zero-Based Budgeting (ZBB)

Every budget line starts at zero. Every expenditure must be justified from scratch against the assignment objectives.

**When to use in proposals:** When the client expects full transparency on every cost component, or when the assignment is novel with no precedent budget.

**When to recommend to clients:** When an organisation's spending has drifted from its strategic priorities, or when eliminating "spend it or lose it" behaviour.

**Process:**
1. Start every category at zero
2. Require a written justification for each expenditure
3. Review each justification against mission, vision, and goals
4. Approve only expenditures aligned with objectives
5. Deny or defer misaligned items

**Advantages:** Eliminates legacy spending; forces alignment with objectives; prevents year-end spending rush.

**Disadvantages:** Time-consuming; requires disciplined justification process; some staff may avoid participating.

### 1.2 Incremental Budgeting

Takes the previous period's budget and adjusts by a fixed percentage or amount.

**When to use in proposals:** For repeat assignments where the firm has historical cost data from similar projects.

**When to recommend to clients:** For stable operational budgets where expenditure patterns are predictable.

**Process:**
1. Take last period's allocation per line item
2. Apply an adjustment factor (inflation, volume change, efficiency gain)
3. Review for any structural changes

**Advantages:** Quick; requires minimal effort; predictable.

**Disadvantages:** Perpetuates misallocation; no scrutiny of whether legacy spending is still justified; encourages "spend it or lose it" behaviour.

### 1.3 Hybrid Approach (Recommended)

Combine methods based on expenditure type. The optimal budget methodology is situational.

**Practical rule:** Use incremental budgeting for predictable, recurring costs (office supplies, routine travel). Use zero-based budgeting for significant, non-routine expenditures (technology procurement, major training programmes, new workstreams).

**Proposal application:** State the methodology used: "Our budget is constructed using a hybrid approach — recurring operational costs are estimated from comparable recent assignments, while technical and specialist costs are built from first principles using zero-based estimation."

## 2. Cost Classification Framework

Seven types of costs (Finkler et al., 2013). Use this taxonomy in proposals and client advisory to demonstrate structured cost thinking.

| Type | Definition | Proposal Example |
|---|---|---|
| **Direct Cost** | Traceable to production of a specific deliverable | Consultant fees for Phase 2 analysis |
| **Indirect Cost** | Not readily traceable to a specific deliverable | Project management overhead, office costs |
| **Variable Cost** | Changes with volume of work | Per diem for additional field visits |
| **Fixed Cost** | Does not change with volume | Software licence, office rental |
| **Semi-Variable Cost** | Partially varies with activity level | Communications (base cost + usage) |
| **Marginal Cost** | Cost of one additional unit | Adding one more training participant |
| **Average Cost** | Total costs divided by units served | Cost per beneficiary trained |

**Proposal application:** Classify costs explicitly in the budget narrative: "Professional fees represent direct costs; project management overhead (estimated at 10% of professional fees) is an indirect cost allocated across all phases."

### The 78% Rule (Public Sector)

In public sector organisations, approximately 78% of the budget goes to personnel costs. After fixed costs (operations, maintenance, facilities), only about 3% is truly discretionary. When advising public sector clients on budgeting, always identify the discretionary margin before proposing new programmes.

## 3. Progressive Cost Estimation Methods

Five levels of cost estimation, from roughest to most precise (Bielefeld). Match the method to the planning stage — never use a detailed method when planning information is insufficient.

### Level 1: Volume/Area-Based Estimation (Roughest)

**When to use:** Very early project phases; feasibility studies; no detailed scope available.

**Formula:**
```
Reference Unit × Cost Parameter = Expected Cost
```

**Three calculation scenarios:**

*Scenario A — Scope specified, estimate cost:*
```
Scope units × cost per unit = expected cost
e.g., 3,000 beneficiaries × $45/beneficiary = $135,000
```

*Scenario B — Budget fixed, estimate achievable scope:*
```
Fixed budget / cost per unit = achievable scope
e.g., $100,000 / $45 = ~2,222 beneficiaries
```

*Scenario C — Both specified, check feasibility:*
```
Fixed budget / required scope = implied cost per unit
e.g., $100,000 / 3,000 = $33/beneficiary
```
Compare against reference range. If below minimum viable cost, the project is not feasible as specified.

**Critical rule:** Always present upper and lower bounds, never a single figure. Show the fluctuation range.

### Level 2: Component-Based Estimation

**When to use:** After the scope of work is defined; phases and deliverables are identified.

**Process:**
1. List each deliverable or work package
2. Estimate the effort (person-days) for each
3. Apply day rates
4. Add non-labour costs per component
5. Sum for total

**Advantage:** Direct connection between cost drivers and cost figures.

### Level 3: Activity-Based Estimation

**When to use:** Detailed methodology is available; activities within each phase are defined.

**Process:**
1. List every activity within each phase
2. Identify the skill/expertise required for each activity
3. Estimate duration and effort for each
4. Apply rates by expertise level
5. Add materials, travel, and other direct costs per activity
6. Sum by phase, then total

**Advantage:** Can be re-sorted by phase (for the client) or by cost category (for the financial proposal).

### Level 4: Unit Price Estimation

**When to use:** When comparable assignments provide reliable unit costs.

**Sources of unit prices:**
- Firm's own database from completed assignments
- Published rate ceilings (World Bank, donor frameworks)
- Market rate surveys
- Contractor quotations

### Level 5: Bottom-Up Work Specification (Most Precise)

**When to use:** All qualities, requirements, and quantities are finalised.

**Process:**
1. Break each activity into individual service items
2. Include ancillary items (preparation, quality checks, documentation, handover)
3. Insert unit prices for each item
4. Multiply by quantities
5. Sum items into activities; activities into phases; phases into total

**Advantage:** Most realistic cost statement; anticipates the work breakdown needed for implementation.

**Practical guidance:** Use a combination of methods — detailed estimation for well-understood components, component-based for less defined areas. Cross-check results from multiple methods.

## 4. The PCTS Constraint Model (Lewis)

Four project constraints — understanding this model is essential for pricing negotiations with clients.

```
C = f(P, T, S)
```

- **P** = Performance (quality requirements)
- **C** = Cost
- **T** = Time
- **S** = Scope (magnitude of work)

**Key principle:** You can fix three constraints; the fourth must be whatever the relationship dictates. Attempting to dictate all four guarantees failure.

**Priority hierarchy:** Performance is typically the highest priority. If late, over budget, or reduced scope — but the deliverable works — the firm will be forgiven. If it does not work, nothing else matters.

**Budget negotiation response:** When clients challenge the price: "Tell me what you can afford, and I'll tell you what I can do" — adjusting scope or extending timeline to match the budget.

### The Time-Cost Trade-Off

There exists an optimum project duration where costs are minimised. Deviations in either direction increase costs:

- **Extending duration** beyond optimum: rising costs from inefficiency, overhead accumulation
- **Compressing duration** (crashing): exponentially rising costs from overtime, coordination overhead, rework

**Proposal implication:** Do not compress timelines to win on price — the actual costs will exceed the quoted amount, creating delivery risk.

## 5. Cost Contingency and Management Reserve (Lewis)

Three components of a well-structured project budget:

| Component | Purpose | Management Rule |
|---|---|---|
| **Work Budget** | Covers cost of identified, planned work | Baseline; tracked through earned value |
| **Management Reserve** | Covers cost of unidentified work (scope changes) | Touched ONLY when new scope is identified; requires formal change control |
| **Margin** | Intended profit | Poor performance eats into margin, not reserve |

**Critical rules:**
- Management reserve is NOT for covering poor estimates or poor performance
- A log must be maintained of all scope changes and their effect on work budget, reserve, and margin
- Money transfers from reserve to work budget must be documented and approved

**Proposal application:** Include a contingency line (typically 5-10% of professional fees) and state its purpose: "A contingency of 7% is included to cover scope adjustments that may be identified during inception. Any use of contingency will be documented and approved by the client through the change control process."

## 6. Contractor and Consultant Pricing Build-Up (Bielefeld)

How to construct a day rate or fee from first principles — useful when clients ask for rate justification.

```
Individual Partial Service Costs (wages + equipment + materials)
+ Site/Assignment Overheads (travel, accommodation, insurance, coordination)
+ General Business Costs (office, management, legal, accounting — typically 15-25%)
+ Risk and Profit margin (typically 10-20%)
= Net fee / day rate
+ VAT
= Gross fee / day rate
```

**Wage calculation:**
```
Effective day rate = Total annual employee costs / Effective working days per year
```

Effective working days = calendar days minus weekends, public holidays, leave, sick days, training days. Typically 200-220 days per year.

## 7. Risk-Adjusted Budgeting (Bielefeld)

### Risk Budget Calculation

For identified risks, calculate the risk budget using the square-root-sum-of-squares method:

```
Risk Budget = √(C₁² + C₂² + C₃² + ...)
```

Where C = individual risk cost (effect × probability).

**Example:**

| Risk | Cost Effect | Probability | Risk Cost |
|---|---|---|---|
| Key expert unavailable — replacement recruitment | $15,000 | 20% | $3,000 |
| Data collection requires additional field visits | $8,000 | 35% | $2,800 |
| Client requests scope extension | $25,000 | 15% | $3,750 |
| Currency depreciation | $12,000 | 10% | $1,200 |

```
Risk Budget = √(3,000² + 2,800² + 3,750² + 1,200²) = √(9M + 7.84M + 14.06M + 1.44M) = √32.34M ≈ $5,688
```

This is more accurate than simply summing risk costs (which would be $10,750) because it accounts for the improbability of all risks materialising simultaneously.

### Buffer Scheduling (Bielefeld)

Identify cost modules that can serve as buffers if risks materialise:

| Cost Module | Full Cost | Reduced Option | Saving | Available Until |
|---|---|---|---|---|
| Stakeholder workshops (3 regional) | $18,000 | 1 central workshop + virtual | $10,000 | Week 6 |
| Printed report (200 copies) | $5,000 | Digital distribution only | $4,200 | Week 10 |
| International benchmarking visit | $12,000 | Desk-based benchmarking | $10,500 | Week 4 |

**Principle:** Compare risk exposure against available buffers on a timeline, exposing gaps in cover.

## 8. Life Cycle Costs (Bielefeld — ISO 15686-5)

For assignments involving infrastructure, systems, or long-lived assets, always present life cycle costs, not just initial investment.

**Life Cycle Cost components:**
- **Initial investment** (design, procurement, implementation, training)
- **Operating costs** (staffing, energy, supplies, licences, hosting)
- **Maintenance costs** (support contracts, upgrades, patches, replacements)
- **End-of-life costs** (decommissioning, data migration, disposal)

**Key principle:** Running costs over an asset's lifetime substantially exceed the initial investment. Higher initial investment in quality can achieve significant lifetime savings.

**Proposal application:** For ICT or infrastructure assignments, include a 5-year TCO table alongside the project budget. This demonstrates strategic thinking and positions the firm as an adviser, not just a deliverer.

## 9. Financial Ratio Analysis (Weiler)

When consulting assignments involve financial assessment, organisational review, or institutional capacity evaluation, use these ratios.

### Liquidity Ratios

| Ratio | Formula | Target | Signal |
|---|---|---|---|
| **Current Ratio** | Unrestricted Current Assets / Unrestricted Current Liabilities | 2:1 | Below 2:1 = cash reserves depleting |
| **Quick Ratio** | (Current Assets − Inventories) / Current Liabilities | 1:1 | Near zero = serious cash flow problem |
| **Available Funds Ratio** | Cash and Short-Term Investments / Current Liabilities | 0.75:1 | Most conservative measure |

### Operating Performance Ratios

| Ratio | Formula | Signal |
|---|---|---|
| **Net Operating Ratio** | (Operating Revenues − Operating Expenses) / Total Revenues | Below 1.0 for 2+ consecutive years = warning |
| **Programme/Function Ratio** | Expenditure by Programme / Total Expenditures | Reveals alignment between stated priorities and actual spending |

### Debt and Solvency Ratios

| Ratio | Formula | Target |
|---|---|---|
| **Debt-Equity Ratio** | Expendable Net Assets / Long-Term Debt | 1:1 or higher |
| **Debt-Service Ratio** | Debt Service / Total Revenues | 8-10% |
| **Contribution Ratio** | Single Revenue Source / Total Expenditures | Diversification check — over-reliance on one source is a risk |

**Warning signals:** Deficit in more than 2 of last 5 years; deficit larger than 10% of total revenue in any single year.

**Cautions:** Never rely on a single ratio. Always use multiple years and trend analysis. Combine financial ratios with narrative context.

## 10. Fiscal Conservatism and Cash Reserve (Weiler)

```
Underestimating Revenues + Overestimating Expenditures = Fiscal Conservatism
```

### Cash Reserve Strategy
1. Maintain 7-10% of total programme value as reserve
2. Spend down reserve progressively through the year to benefit current-year beneficiaries
3. Understand fund-specific restrictions — some reserves have strict use requirements
4. Seek shared funding from other departments or budget lines for unanticipated costs

**Proposal application:** When advising clients on budget frameworks, always recommend a reserve policy. In financial proposals, include a contingency that demonstrates fiscal conservatism without inflating the price.

## 11. Time Value of Money (Weiler)

Essential for capital budgeting, cost-benefit analysis, and multi-year programme design.

### Present Value
```
PV = FV / (1 + r)ⁿ
```

### Future Value
```
FV = PV × (1 + r)ⁿ
```

### Net Present Value
```
NPV = Σ [discounted (annual inflows − annual outflows)] − initial outlay
```
**Decision rule:** Reject projects with negative NPV (purely financial decisions).

### Choosing the Discount Rate (Dunn, 2012)
| Approach | Basis | Limitation |
|---|---|---|
| **Private rates** | Private sector returns | May not represent public interest |
| **Social rates** | Collective value for future generations | Often criticised as inefficiently low |
| **Government rates** | Current government borrowing costs | Ignores opportunity cost |
| **Inflation-adjusted** | Include anticipated inflation in the rate | Requires reliable inflation forecasts |

**Proposal application:** For assignments involving feasibility studies, options appraisal, or investment justification, always include NPV analysis. State the discount rate used and the rationale for choosing it.

## 12. Forecasting Methods (Weiler)

For proposals that include demand projections, revenue forecasts, or expenditure predictions.

### Moving Average
```
Forecast = Sum of values for included periods / Number of periods
```
More periods = greater smoothing. Fewer periods = more responsive to recent changes.

### Weighted Moving Average
Assigns explicit weights to time periods (weights must sum to 1.0):
```
Forecast = w₁ × Value₁ + w₂ × Value₂ + w₃ × Value₃
```

### Exponential Smoothing
```
Ft = Ft₋₁ + α(At₋₁ − Ft₋₁)
```
Where α (smoothing constant) ranges from 0.05 to 0.50. Lower α = assumes stability; higher α = more reactive.

### Linear Regression (OLS)
```
Yt = a + b(t)
```
Minimum 20-30 observations needed for valid forecasting. Must verify linear relationship exists before applying.

**Proposal application:** When a methodology involves forecasting (demand projection, revenue estimation, population growth), name the method and justify the choice. Reference the minimum data requirements.

## 13. Budget Alignment and Oversight

### Strategic Alignment Test (Weiler)

Every expenditure should align with four documents:
1. **Mission** — what the organisation does now
2. **Vision** — what it strives to become
3. **Goals** — short- and long-term targets
4. **Strategic plan** — specific actions over a predetermined period

**Alignment audit process:**
1. State the values from mission/vision/goals/strategic plan
2. Identify observable values from actual spending patterns
3. Compare stated vs. observable values
4. Assess alignment quality
5. Correct misalignment through coaching or systemic change

**Proposal application:** When advising clients on budgeting, always include an alignment assessment. A budget that does not serve the strategy is a budget that wastes resources.

### Variance Analysis

Two types of budget variances:

| Type | Cause | Response |
|---|---|---|
| **Cyclical** | Anticipated or seasonal fluctuations | Budget stabilisation; smooth across periods; use reserves |
| **Structural** | Changes to budget structure, policy, or rules | Alter remaining periods; make cuts in flexible areas; treat as a transition |

### Cost Control Sequence (Bielefeld — per cost category)

1. Fix the budget for the category
2. Obtain quotations or estimates
3. Compare estimates against budget — flag deviations immediately
4. Monitor actual spending during execution
5. Track all changes in scope or specification
6. Reconcile final costs against budget

**Decision and Change Log:**

| # | Date | Decision/Change | Approved | Cost Impact |
|---|---|---|---|---|
| 1 | [Date] | [Description] | Yes/No | +/− [Amount] |

**Principle:** Every cost-relevant decision must be assessed and communicated immediately, not summarised later. This enables the client to compensate by adjusting scope, quality, or resources elsewhere.

## 14. Six-Stage Costing Decision Gates (Bielefeld)

For assignments involving project costing or procurement advisory, this framework shows how cost estimates should evolve through a project lifecycle.

| Stage | Decision Point | Method | Purpose |
|---|---|---|---|
| 1. Cost Framework | Go/no-go on project | Volume/area parameters | Check statistical realism |
| 2. Cost Estimate | Approve preliminary design | Component estimation | Compare design alternatives |
| 3. Cost Calculation | Submit for approval | Detailed element costing | Create "facts" via formal approval |
| 4. Cost Specification | Go to tender | Construction elements reformulated for procurement | Crucial — limited scope for changes after |
| 5. Price Comparison | Award contract | Tender evaluation | Compare bids item by item |
| 6. Cost Establishment | Determine actual costs | Sum of final invoices | Feeds future cost databases |

**Proposal application:** When an assignment involves procurement or capital budgeting, reference this staged approach to demonstrate that cost management is progressive and disciplined, not a one-time estimate.

## 15. Building a Cost Database

Derive cost figures from completed assignments to improve future estimates:

```
Cost Figure = Invoiced Sum / Quantity Delivered
```

**Best practices:**
- Store figures excluding VAT (tax rates change)
- Record the year so values can be adjusted for inflation
- Build your own database from your own projects — significantly more precise than published averages
- Include all ancillary costs when deriving figures

**Proposal application:** Reference the firm's cost database as an asset: "Our estimates are benchmarked against our database of [N] comparable assignments delivered in the East African market over the past [N] years."

## 16. The "Wants vs. Needs" Framework (Gordon)

A simple but powerful framework for budget prioritisation, applicable in both proposal scoping and client advisory.

- **Need:** Something the project cannot function without
- **Want:** Something desirable but not essential

**Application in proposals:** Categorise deliverables as core (essential to the ToR) vs. optional (value-adding but not required). Present a tiered pricing structure:
- **Option A:** Core scope — $[amount]
- **Option B:** Core + enhanced scope — $[amount]

This gives the client pricing flexibility without compromising the core assignment.

**Application in client advisory:** When clients face budget constraints, apply the needs/wants analysis to prioritise expenditure. Maps directly to MoSCoW prioritisation (Must have, Should have, Could have, Won't have this time).

## 17. The "Cut Back, Not Out" Principle (Gordon)

When budgets must be reduced, reduce amounts across categories rather than eliminating entire workstreams.

**Proposal application:** When a client asks for a lower price, reduce scope items proportionally rather than dropping entire phases. Maintain the overall structure of the assignment while adjusting the depth of each component.

**Client advisory application:** When advising organisations on budget cuts, present graduated reduction options rather than binary cut/keep decisions. Show the impact of each reduction level on service delivery.
