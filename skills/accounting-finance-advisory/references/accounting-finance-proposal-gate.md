# Accounting Finance Proposal Gate

Use this gate when building or reviewing proposal sections for accounting, finance systems, controls, or financial modelling work.

## Methodology Workstreams

| Workstream | What to include |
|---|---|
| Current-state finance review | Source documents, ledger quality, system setup, reporting pack, close process, pain points |
| Transaction walkthroughs | Sales-to-cash, procure-to-pay, inventory-to-COGS, payroll-to-payment, tax-to-return, capex-to-depreciation |
| Reconciliations | Bank/mobile money, AR, AP, inventory, payroll, tax, fixed assets, loans, grants, intercompany |
| Controls review | Roles, approvals, maker-checker, audit logs, period locks, reversal workflow, document retention |
| Management accounting | Cost centers, profit centers, branch/project/product profitability, budgets, variance, CVP, KPIs |
| Systems design | Chart of accounts, posting rules, dimensions, workflows, integrations, migration, reports, dashboards |
| Financial model assurance | Inputs, calculations, outputs, checks, scenarios, assumptions, version control, audit trail |
| Advanced reporting | IFRS/local GAAP gap review, fair-value/impairment support, consolidation/intercompany logic, loan/lease schedules |
| Handover | SOPs, training, close checklist, reconciliation templates, report pack, issue log, roadmap |

## Proposal Deliverables

- Finance diagnostic report with findings, impact, root cause, and recommendations.
- Accounting requirements specification or ERP/POS finance configuration design.
- Chart of accounts and reporting-dimension redesign.
- Posting-rule register for core business events.
- Reconciliation and close pack.
- Controls matrix with roles, approvals, evidence, and exception handling.
- Financial model build or audit report with scenario/sensitivity outputs.
- Management accounts and dashboard specification.
- Consolidation, intercompany, fixed asset, loan, lease, or grant-accounting workpapers where the assignment requires them.
- Training plan and finance SOPs.

## Pricing Drivers

- Number of entities, branches, currencies, products, users, and systems.
- Transaction volume and period covered.
- Number of bank/mobile money accounts.
- Inventory SKUs, warehouses, costing complexity, and stock-count effort.
- Payroll headcount, tax complexity, and statutory filing needs.
- Data cleanup, migration, reconciliation backlog, and Excel model risk.
- Workshops, interviews, onsite observations, validation cycles, and reporting formats.

## Red Flags To Fix Before Submission

- Proposal promises "automation" but omits posting rules, reconciliations, and controls.
- Budget excludes data cleanup, transaction sampling, chart-of-accounts review, or validation workshops.
- Work plan has no period-close, reconciliation, or user-acceptance phase.
- Financial proposal prices the work as generic IT while technical scope requires accounting expertise.
- Deliverables are named broadly but have no acceptance criteria.
- Tax, statutory, donor, or audit evidence requirements are ignored.

## Acceptance Criteria

- A reviewer can see exactly how the assignment will improve financial accuracy, controls, reporting, and decision-making.
- The financial proposal includes enough level of effort for accounting review, data work, validation, and training.
- The methodology can survive questions from a CFO, accountant, auditor, donor finance officer, or procurement evaluator.
