# AI-Agent Win Themes and Discriminators

Reusable win themes, discriminators, and ghost-pack language for AI-agent bids. Pairs with `ai-agent-vertical-positioning` and the proposal section skills.

## Top Win Themes

### Win Theme 1 — Autonomy is committed, not assumed
- Autonomy is committed per action class; the agent never approves an irreversible action on its own.
- The buyer remains accountable to the regulator and the customer; the agency carries delivery liability.
- Trust staging from Shadow → Supervised → Agentic with binary evidence at each gate.

### Win Theme 2 — Action catalogue is the contract
- Every action the agent can take is enumerated, classified by reversibility, bounded in value and parameters, rate-limited.
- The catalogue is signed; changes require a change order.
- Procurement, audit, and the regulator can read what the agent will do before launch.

### Win Theme 3 — Kill-switch is real, drilled, and named
- Four layers — per-action, per-agent, per-tenant, global — with named authority and after-hours coverage.
- Time-to-stop SLA: 60 seconds (per-action / per-agent); 5 minutes (per-tenant); 15 minutes (global).
- Quarterly drills; drill log available to the buyer's auditor.

### Win Theme 4 — Full audit log to regulator-grade
- Every agent action logged with reasoning trace, tool call, parameters, result, human decision.
- Completeness SLA 99 %; retention seven years or sectoral requirement.
- Replay-deterministic for forensic review.

### Win Theme 5 — Supervisor is reskilled, not just renamed
- Funded supervisor retraining curriculum.
- New operating model with named authority.
- Adoption metrics include supervisor CSAT and queue headroom, not just usage.

### Win Theme 6 — Transparency-to-affected-party is engineered
- Every external communication discloses the agent in line with applicable rules.
- Voice agents identify at the start of the call.
- Contestability channel published; redress within SLA.

### Win Theme 7 — Sovereign-AI is a real option
- Open-weight model on dedicated GPU; on-prem inference; in-country audit log with mirror to buyer SOC.
- Sectoral procurement (financial services, public sector, healthcare) supported.

### Win Theme 8 — Honest economics
- Tasks-per-FTE saved with intervention discount, not without.
- Cost per outcome at month 1 and month 6, not just month 6.
- Three scenarios keyed to autonomy ramp, not just demand.
- Downside includes irreversible-incident dollar figures and regulator action.

## Discriminators

| Discriminator | Why it wins | Where competitors fail |
|---|---|---|
| Action Catalogue signed before build | Procurement and audit can read it; change-control disciplined | Competitors deliver an agent and post-hoc rationalise scope |
| Reversibility classification on every action | Irreversibility incidents impossible by design | Competitors handle reversibility as a UX concern |
| Autonomy committed per action class | The same agent can be safe at L3 on triage and L1 on action | Competitors quote a single autonomy level per agent |
| Quarterly kill-switch drill with log | Operational reality, not slide-deck commitment | Competitors describe the kill-switch but never drill |
| Component-level eval | Failure isolated; not hidden in end-to-end pass | Competitors run only end-to-end eval |
| Red-team includes prompt injection via tool output | Catches the under-addressed agent attack class | Competitors red-team prompt injection only on direct user input |
| Funded supervisor retraining | Adoption is engineered | Competitors expect supervisors to figure it out |
| Transparency-to-affected-party UX | Survives Article 50 and sectoral rules | Competitors hide the agent |
| Sovereign-AI option | Public sector and regulated banks accept the deployment | Competitors offer one cloud only |
| Honest tasks-per-FTE with intervention discount | CFO trusts the case | Competitors quote month-12 numbers as if month-1 |

## Ghost-Pack Language

### Disqualifying competitors who over-promise autonomy
> Some vendors offer fully autonomous decisioning on regulated workflows. Our methodology refuses to do so. The agent assembles; a named human decides. The buyer remains accountable to the regulator; we will not promise otherwise.

### Disqualifying competitors who hide the agent
> Some vendors deploy agents that pose as humans in customer communications. This breaches EU AI Act Article 50 and similar disclosure rules and erodes customer trust. Our agents disclose; the contestability channel is always present.

### Disqualifying competitors who skip the kill-switch drill
> Some vendors describe a kill-switch but never test it. We drill quarterly with a written log available to the buyer's auditor and, where invited, the regulator.

### Disqualifying competitors who quote unmoderated autonomy savings
> Some vendors quote tasks-per-FTE saved as if the agent achieves target autonomy on day one. Our model applies the intervention discount honestly — at month 1, at month 6, at month 12 — so the CFO sees the curve, not the destination.

### Disqualifying competitors who quote per-resolution without definition
> Some vendors quote per-resolution pricing with "resolution" undefined. Our definition is explicit — closure without reopen for N days — and our pricing includes intervention credit if the rate exceeds ceiling.

### Disqualifying competitors who rename roles without reskilling
> Some vendors rename customer-support agents to "supervisors" without changing the operating model. We deliver a funded six-module curriculum, define the supervisor's new authority, and track supervisor CSAT and queue headroom as adoption metrics.

## Sample Executive-Summary Differentiator Block

> Four discriminators set our agentic delivery apart. First, autonomy is committed per action class — the agent never approves an irreversible action on its own. Second, the action catalogue is signed and the kill-switch authority is named with quarterly drills and a written log. Third, the supervisor's role is reskilled with a funded curriculum, not renamed on a slide. Fourth, transparency-to-affected-party is engineered into every external communication so the agent never poses as a human and the contestability channel is always present.

## How to Use

- Pick three to five themes per bid; do not list ten.
- Match the discriminators to the vertical (`ai-agent-vertical-positioning`).
- Use ghost-pack language where the competitor is named or strongly inferred; otherwise use the disqualifying patterns as positive statements.
- Cross-reference to evidence in methodology, risk register, and compliance section.
