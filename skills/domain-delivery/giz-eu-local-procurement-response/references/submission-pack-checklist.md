# GIZ Submission Pack Checklist

Parent skill: [GIZ EU Local Procurement Response](../SKILL.md).

Use this as the final mechanical check before a GIZ submission leaves the control room.

## Technical Folder

- Correct transaction number in folder/file names.
- Technical concept within page and font limits.
- CVs within CV page limits and role/duration evidence included.
- All mandatory declarations included where required.
- No price, rate, budget, subtotal, total, VAT, remuneration, fee-day cost, travel cost, or financial formula appears.
- File opens successfully after zipping.
- File size within tender limit.
- Every scored criterion has a visible answer.

## Financial Folder

- Correct transaction number in folder/file names.
- Price schedule uses the buyer's template without unauthorised edits.
- All rates and totals are net of VAT where required.
- Person-days reconcile to the technical staffing narrative.
- Fixed budgets are treated exactly as instructed.
- Formulae and totals have been checked independently.
- File opens successfully after zipping.

## Template Integrity

- No protected wording deleted.
- No extra terms added to the buyer's forms.
- No hidden comments or tracked changes remain.
- No stale draft labels remain.
- No external links carry load-bearing information excluded by the tender rules.

## Final Release Gate

Before sending:

1. Compliance matrix shows no open mandatory item.
2. Eligibility verdict is pass or documented consortium solution.
3. Technical folder has zero price tokens.
4. File-size and page limits pass.
5. A second reviewer signs off.
6. Sending address, subject line, deadline, and attachment list match the tender.
