# GIZ Compliance Matrix

Use this reference when building the mandatory and scored matrix for a GIZ local tender.

## Matrix Fields

| Field | Purpose |
|---|---|
| Requirement ID | Stable internal ID for traceability. |
| Tender source | Annex, section, page, or form where the requirement appears. |
| Requirement text | Verbatim or tightly paraphrased requirement. |
| Type | Mandatory, scored, eligibility, format, submission, contractual, financial. |
| Weight/points | Points or percentage if scored. Use blank for pass/fail. |
| Evidence required | Document, form, CV item, register extract, reference, calculation, declaration. |
| Owner | One accountable person. |
| Location in bid | File and section where evidence appears. |
| Status | Open, partial, covered, blocked, not applicable. |
| Risk | Low, medium, high, knockout. |
| Reviewer sign-off | Name/date of reviewer. |

## Knockout Tests

Run before drafting:

- Legal registration and authorised signatory are evidenced.
- Registration extract is within the age limit required by the tender.
- Sanctions and exclusion self-declaration can be signed truthfully.
- Minimum average turnover is met for the required years.
- Minimum staff/managers threshold is met.
- Reference projects meet value, date, field, geography, and role requirements.
- Tender templates can be completed without alteration.
- Submission channel, file-size, language, font, page, and deadline constraints are achievable.

## Scoring Alignment

For every positively weighted criterion:

1. Extract the assessor's criterion language.
2. Identify the evidence that will make the claim credible.
3. Allocate page budget according to score weight.
4. Link the criterion to the CV/profile proof if personnel carries most points.
5. Mark any criterion that cannot be proved from team or reference evidence as a bid risk.
