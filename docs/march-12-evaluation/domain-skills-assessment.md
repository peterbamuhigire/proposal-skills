# Supporting Domain Skills Assessment

**Date:** 12 March 2026

---

## What Domain Skills Do

Domain skills are cross-cutting knowledge bases that proposal sections draw from. When a methodology section needs a change management approach, section 06 calls change-management/SKILL.md. When an M&E plan is required by the ToR, it generates a standalone section. These skills must be deep enough that Claude can produce content matching what a Big Four firm's subject matter expert would write.

### Project Management (145 + 356 = 501 lines) — GOOD

**SKILL.md covers:**
- PRINCE2 (stage-gate, product-based planning)
- PMBoK (5 process groups, 10 knowledge areas)
- Agile/Hybrid approaches
- Governance (steering committee, PMO)
- Reporting standards (weekly, monthly, inception report)
- Escalation framework (3-level matrix)
- Earned value management (PV, EV, AC, SPI, CPI)
- Change control and lessons learned

**Reference file covers:**
- Lewis Method (10-step PM)
- WBS construction
- PERT three-point estimation
- CPM/critical path
- Full earned value system (SV, CV, SPI, CPI, CR, EAC)
- 15% rule for early warning
- PM maturity model
- Stoplight reporting

**What's missing for world-class:**
- Schedule risk analysis and Monte Carlo simulation
- Earned Schedule methodology (beyond earned value)
- Probabilistic scheduling
- Quality gates tied to deliverable acceptance
- Servant leadership and team dynamics
- Conflict resolution protocols
