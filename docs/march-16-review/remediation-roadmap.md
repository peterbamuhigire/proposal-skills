### P2.2 — Evaluation Scoring Guidance (Gap C3)

**Priority:** High — directly improves point allocation
**Effort:** 0.5 session, ~150 lines
**Output:** 10 edits to section SKILL.md files

Add an "Evaluation Scoring" subsection to every section SKILL.md with:
- Typical weight range by procurement framework (WB QCBS, PPDA, UNDP, AfDB)
- What evaluators prioritise in this specific section
- Common point-losing mistakes
- How to maximise score within page/time constraints

**Sources needed:** None. This codifies existing knowledge from the procurement framework references.

### P2.3 — Cross-Section Theme Threading (Gap C9)

**Priority:** High — differentiates from mid-tier competitors
**Effort:** 1 session, ~100 lines
**Output:** ~6 edits across parent SKILL.md and section skills

Mechanism:
1. Parent SKILL.md: require 3-4 win themes to be defined before writing any section
2. Section 02 (executive summary): theme definition guidance
3. Each section SKILL.md: "Theme Integration" note showing how themes manifest

**Sources needed:** None. Draws on existing persuasion and patterns references.

### P2.4 — Weak-vs-Strong Calibration Examples (Gap C4)

**Priority:** Medium-high — improves output quality
**Effort:** 1 session, ~300 lines
**Output:** 4-10 edits to section SKILL.md files

Focus on sections where quality variance is highest:
- 02 (executive summary) — weak generic vs strong quantified
- 03 (understanding of assignment) — parroting ToR vs demonstrating insight
- 05 (relevant experience) — activity-focused vs outcome-focused project cards
- 06 (methodology) — generic phases vs named frameworks with rationale

**Sources needed:** None. Examples can be crafted from professional experience.

**Effort:** 4 sessions, ~1,500 lines
